# Specs Index

> **Note:** The `specs/` directory contains early-stage product planning artifacts developed during the initial phases of this project. These files were used to explore the scope of a potential food surplus marketplace — including consumer and merchant UX flows, payment flows, and full marketplace architecture. They represent exploratory design thinking, not the delivered product. The actual submission deliverables are listed in the README.

| File                     | Domain      | Description                                                     |
| ------------------------ | ----------- | --------------------------------------------------------------- |
| data-model.md            | Data        | All entities, relationships, constraints, indexes               |
| authentication.md        | Auth        | User registration, login, roles, session management             |
| marketplace.md           | Marketplace | Listing creation, search, filtering, purchase flow              |
| pricing.md               | Pricing     | Dynamic pricing engine, price curves, merchant controls         |
| ml-surplus-prediction.md | ML          | Surplus prediction models, features, serving                    |
| ml-recommendations.md    | ML          | Recommendation engine, ranking, cold start                      |
| ml-analytics.md          | ML          | Waste analytics, pattern detection, prescriptive insights       |
| merchant-experience.md   | UX          | Merchant dashboard, listing management, order fulfillment       |
| consumer-experience.md   | UX          | Consumer app, deal discovery, purchase, pickup, impact tracking |
| payments.md              | Payments    | Payment processing, refunds, payouts, QR pickup                 |
| notifications.md         | Comms       | Push notifications, alerts, timing optimization                 |
| food-safety.md           | Compliance  | SFA regulations, expiry tracking, safety verification           |
